# LINE bot + AI自動応答 要件定義書

模擬案件1（SKH講座 実践編CHAPTER 2 初級）
クライアント: 個人経営美容室オーナー（40代女性）／CrowdWorks経由の依頼

## 1. 背景と本質的な課題

問い合わせ対応に時間を取られて収益を生む施術に集中できていないので、よくある質問への対応を自動化し、施術に使える時間を取り戻したい。

## 2. ヒアリングで確定した事実

| 項目 | 回答 |
|---|---|
| 問い合わせ件数 | 1日10〜15件（土日はやや多い） |
| よくある質問トップ5 | 営業時間／カットの値段／予約取れますか／パーマとカラー同時にできる？／駐車場ある？ |
| 料金改定の頻度 | 半年に1回程度の値上げあり → FAQは自分で更新できる必要 |
| botの範囲 | まずは質問応答のみ。予約は将来的に |
| LINE公式アカウント | 保有済み。友だち約300人 |
| 答えられない質問 | オーナーに通知してほしい |
| 予算 | 初期10万円以内・月額はなるべく安く |
| 追加要望 | 臨時休業などのお知らせ一斉配信もしたい |

## 3. スコープ（3分解）

### やること（本プロジェクト）
1. **FAQ自動応答**: 頻出5質問（営業時間／メニュー・料金／アクセス・駐車場／施術の組み合わせ 等）にAIが回答
2. **エスカレーション通知**: botが答えられない質問はオーナーのLINEに通知
3. **管理画面**: FAQ・メニュー料金をスマホで更新できる（PC不要）
4. **お知らせ一斉配信**: 臨時休業などを友だち約300人に一斉送信

### やらないこと（スコープ外）
- 予約機能（電話・既存LINEでの手動対応を継続）
- 顧客カルテ・決済機能
- 多言語対応

### 将来やること（Phase 2提案）
- 予約機能の追加（カレンダー連携・空き状況の自動回答）
- リピーター向けパーソナライズ応答

## 4. 技術構成

```
LINE Messaging API → Webhook → Next.js API Route → Claude API
                                  ↓
                        Supabase（FAQ／メニューDB）
                                  ↓
                        管理画面（Next.js）
```

| 層 | 採用技術 | 理由 |
|---|---|---|
| フロント/API | Next.js + Vercel | 無料枠で完結／デプロイ自動／Claude Codeとの相性◎ |
| DB | Supabase | 無料枠で十分／RLSでセキュリティ確保 |
| AI | Claude API | 日本語が自然／想定件数なら月数百〜数千円 |
| LINE連携 | LINE Messaging API | 既存の公式アカウント（友だち300人）をそのまま活用 |

## 5. 非機能要件・制約

- 予算: 初期10万円以内／月額なるべく安く（目標5,000円以下）
- ランニングはClaude API従量課金のみ（他は無料枠）
- 注意: LINE公式アカウント無料プランは送信月200通まで（応答メッセージはカウント外）。一斉配信300人=300通/回のため、配信を使う月はLINEライトプラン（月5,000円）が必要。提案書に注記済み
- FAQ更新はオーナー自身がスマホで完結できること（半年に1回の料金改定に対応）
- 機密情報（LINEチャネルシークレット・APIキー）は環境変数で管理し、コードに直書きしない

## 6. 想定効果（提案書の根拠数字）

- 1日10〜15件のうち頻出5質問が大半 → **約8割を自動応答**
- 対応時間: 10〜15件 × 2〜3分 ≒ 1日約30分 → エスカレーション2〜3件のみで **1日5分以下**
- 営業時間外・施術中でも即レス可能に
