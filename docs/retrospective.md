# 振り返りメモ（案件1: LINE bot + AI自動応答）

作成日: 2026-07-19（納品完了時）

## ① 自己評価（3観点）

### 技術面

- **詰まったところ**
  - Supabaseが初回アクセスのみ `PGRST303 JWT issued at future` を返す事象。正体は「一時的な時計ズレ応答がNext.jsのdevキャッシュ(.next)に残って再生されていた」。エラーコードだけ見て原因を推測せず、エラーメッセージ全文を出す→キャッシュ削除で切り分ける、の手順が有効だった
  - Next.js 16でmiddlewareがproxy.tsに改称されていた。学習データより新しいフレームワークは**ローカルの公式docs（node_modules内）を先に読む**ルールが効いた
- **うまくいった工夫**
  - 回答と確信度を構造化出力（JSONスキーマ固定）で1回のAPI呼び出しにまとめた（低コスト・低レイテンシ・パース安定）
  - Webhookは即200→`after()`で本処理、の分離でLINEタイムアウトを構造的に回避（本番実測964ms）
  - 障害時もお客様を無応答にしないフォールバック設計（謝罪文+オーナー通知）
- **次回試したい技術**: LINEリッチメニュー、Flex Message、会話履歴を使ったマルチターン応答

### PM面

- **ヒアリング→スコープ3分解**（やる/やらない/将来やる）が効いた。予約機能を「将来」に切り出したことで初期予算内に収まり、Phase 2提案の種にもなった
- **要件にない発見を先回りできた**: LINE無料プラン200通/月の制約（配信300人=300通）を提案書段階で注記。納品直前だと「聞いてない」トラブルになるところだった
- **反省**: 配信テスト前に友だち数を確認する運用（誤配信リスクの事前確認）を、クライアント役に指摘されてから行った。破壊的操作の前提確認は自分から先にやるべきだった

### Claude Code活用度

- 使った機能: Plan Mode（段階1設計）/ カスタムサブエージェント定義（code-reviewer等3種）/ ブラウザ操作（Anthropic Console・LINE Developers・OAM・Vercel・Supabaseの設定代行）/ バックグラウンドタスク（デプロイ待ち→検証の自動化）/ ローカルdocs参照 / スキル（ソウ=PMロール運用）
- 特に効いたのは**ブラウザなしE2Eテスト**の発想: 署名付きWebhook直POSTとServer Action直POST（$ACTION_IDのプログレッシブエンハンスメント経路）で、UI操作なしに本番の追加→bot反映→削除まで自動検証できた
- 未活用: Worktree並行開発、code-reviewerサブエージェントの毎コミット運用（下記改善メモへ）

- 誤差-50%超（下振れ）。要因は①Claude Codeによる実装・検証・外部サービス設定の代行 ②デプロイを段階1で先行させ「本番で小さく確かめる」を毎段階で回せたこと ③要件が明確でヒアリングの手戻りゼロ
- 教訓: **人力前提の見積もりのままだとAI活用時代の相場から乖離する**。次案件は「Claude Code併用前提の実働」と「クライアント提示用のバッファ込み」を分けて見積もる

## ③ 改善メモ（次の案件に活かす）

- 破壊的・不可逆操作（一斉配信・削除・公開）の前には、**影響範囲の実数確認を自分から先に**行う（今回: Insight APIで友だち数確認）
- 独立した2機能（例: 管理画面と配信）は**Worktreeで並行開発**を試す
- `code-reviewer` サブエージェントを**コミット前に必ず走らせる**運用にする（今回は手動レビューのみ）
- 新しめのフレームワークは**最初にnode_modules内の公式docsの該当ページを読む**（middleware→proxyで実証済み）
- 見積もりは「AI併用実働」と「提示価格」を分離して作る
